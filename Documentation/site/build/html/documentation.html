Documentation¶

Lost in Space : Retrouve une étoile sur une image

Traitement Image¶

class Lostinspace.TraitementImage¶

Classe Traitement Image permettant de réaliser un traitement” d’une image d’un ciel étoilé. Cette image traité peu ensuite être analysée Méthodes : NomImage, CoorEtoileLabel, CoorEtoileGaussian, Affichage, Maxietoile, RapportImage

Affichage()¶

Un affichage de l’image en niveau de gris avec des cercle entourant les zones détecté si une méthode de détection à été utilisé

CoorEtoileGaussian()¶

Permet de connaître la position et la taille de chaque étoiles de l’image analysée Pour se faire on utilise la fonction Gaussian https://en.wikipedia.org/wiki/Blob_detection#The_Laplacian_of_Gaussian

Type renvoyé

list

Renvoie

Liste des des étoiles sous le format (x,y,sigma) position x , position y , sigma / info : sigma*(2)**0.5 envrion égale à rayon

CoorEtoileLabel()¶

Permet de connaître la position et la taille de chaque étoiles de l’image analysée Pour se faire on utilise la fonction Label https://en.wikipedia.org/wiki/Connected-component_labeling

Type renvoyé

list

Renvoie

Liste des des étoiles sous le format (x,y,r) position x , position y , rayon

Maxietoile(liste_etoile, nb=50)¶

Sélectionne les nb plus grandes étoiles (théoriquement les nb plus lumineuse) et les renvoies dans l’ordre décroissant.

Paramètres

  • liste_etoile – list => Liste d’étoiles sous le format (x,y,r/sigma).

  • nb – int => nombre d’étoile a sélectionner dans la liste.

Renvoie

Renvoie une liste des étoiles les plus lumineuses de nb élément dans l’ordre décroissant.

Type renvoyé

list

NomImage(nom)¶

Permet de définir le nom de l’image à analyser. Attention ! ne pas oublier l’extension Cette image serra utiliser pour l’ensemble des fonction de la classe TraitementImage

Paramètres

nom – str => nom de l’image à analyser, exemple “crabe.png” .

RapportImage(liste_étoile)¶

Permet de créer des rapports grace à 3 points formant un triangles les rapports sont petit coté / grand coté et moyen coté / grand coté

Paramètres

liste_étoile – list => Liste d’étoiles sous le format (x,y,r/sigma).

Type renvoyé

dict

Renvoie

Renvoie un dictionnaire contenant un tuple de rapports et leurs erreurs (r1,dr1,r2,dr2) en clé associé à un tuple contenant des trios d’étoiles chacune caractérisée pas des coordonnées cartésiennes (xi,yi,ri/sigmai).

Exemple : dico={(r1,dr1,r2,dr2): ((x1,y1,r1),(x2,y2,r2),(x3,y3,r3))…}

Base De Donee¶

class Lostinspace.BaseDeDonee¶

Classe qui concerne tout ce qui se rapporte à la Base de Données. A savoir, l’importation des coordonnées des étoiles dans la région observée et le calcul des rapports des triangles formées à partir de ses étoiles

Méthodes: Build, ListeN, rapportBaseDeDonnee, Affichage

Affichage()¶

Un affichage de la base de donnees (une image)

BUILD(Adroite, Declinaison)¶

Initialise les coordonnées de l’objet que l’on veut observer en équatoriale = (h,m,s),(°,”,””) = (heure minute seconde) et (Degree Minute d’arc Seconde d’arc) Exemple (5, 34, 31.97),(22, 0, 52.1) Coordonnées nébuleuse du Crabe

Paramètres

  • Adroite – tuple => les coordonnées de l’objet que l’on veut observer heure minute seconde.

  • Declinaison – tuple => les coordonnées de l’objet que l’on veut observer Degree Minute d’arc Seconde d’arc.

ListeN(n, angle=2.5)¶
Paramètres

  • n – int => nombre d’étoile a sélectionner dans la Base de Données.

  • angle – float => valeur du rayon du cercle autour de l’astre à observer pour une image 5°x5° il faut donner 2.5.

Type renvoyé

list

Renvoie

Une liste des étoiles les plus lumineuses dans l’ordre décroissant

format : (x,y,magnitude)

rapportBaseDeDonnee(image)¶

Permet de créer des rapports grace à 3 points formant un triangles les rapports sont : petit coté / grand coté et moyen coté / grand coté

Paramètres

image – list => liste des étoiles au format : (x,y,magnitude).

Type renvoyé

tuple

Renvoie

Deux éléments :

Renvoie un dictionnaire contenant un tuple de rapports (r1,r2) en clé associé à un tuple contenant des trios d’étoiles chacune caractérisée pas des coordonnées gnomoniques (xi,yi).

Exemple : dico={(r1,r2): ((x1,y1),(x2,y2),(x3,y3))…}

Renvoie aussi une liste composée des coordonnées (xi,yi) et magnitude (Mi) des toutes les étoiles de la région observée

Exemple : image=[(x1,y1,M1),(x2,y2,M2)]

Comparaison¶

class Lostinspace.Comparaison¶

Classe qui s’occupe de comparée l’image à la base de donnée

Affichage(image='crabe.png')¶

Un affichage de deux triangles, si les triangles sont similaires c’est que la transformation est réussite, s’ils ne le sont pas cela n’a pas forcément échoué Affiche 2 triangles qui s’ils sont similaires montrent que la transformation est un succès

Paramètres

image – str => image de fond, mettre la même que l’image analysé pour un bon visuel.

AstreCherche(tuplecoo)¶

Permet de décaller le centre de la base de données En 0,0 nous somme centrée

KHI2(enzo, dico, imagetrilumi)¶

Applique la formule du Khi2 afin de comparer les rapports de l’image et ceux de la base de données

Paramètres

  • enzo – dict => Returns de la méthode rapport image dans la class TraitementImage

  • dico – dict => Returns de la méthhode rapportBaseDeDonnee dans la class BaseDeDonnee

Renvoie

2 listes Lbdd et Lima qui contiennent les étoiles de la base de données et de l’image qui correspondent entre elles

Type renvoyé

tuple

transfo(ebdd, eima)¶

Applique des formules d’homothétie à savoir une translation une rotation et une mise à l’échelle afin de déterminer une transformation géométrique qui permettrait de passer de la base de données à l’image.

Paramètres

  • ebdd – list => Correspond à la liste des triangles formés à partir de la base de données dans la méthode transfos.

  • eima – list => Correspond à la liste des triangles formés à partir de l’image dans la méthode transfos.

Renvoie

Eqaution: Dictionnaire contenant les variables de la fonction de transfert, e: les coordonnées de l’astre cherché (en cartésien pixel) , aff: ensemble de données utiles à l’affichage.

Type renvoyé

tuple

transfoetoile(transfo)¶

Applique les tranformation à d’autres points

transfos(lbdd, lima)¶

regroupe les étoiles déterminées par le khi2 par trois afin d’en faire des triangles Aussi bien pour Lbdd que pour Lima.

Paramètres

  • Lbdd – list => Liste regroupant les étoiles de la base de données retenues.

  • Lima – list => Liste regroupant les étoiles de l’Image retenue.

Renvoie

Une liste qui contient la liste des transformations créées à l’aide de la méthode transfo.

Type renvoyé

list

vote(EE)¶

Algorithme de vote classique qui détermine la transformation géométrique la plus réaliste.

Paramètres

EE – list => liste des transformations déterminées à l’aide des méthodes transfos et transfo.

Renvoie

Dictionnaire contenant les variables de la transformation retenue.

Type renvoyé

dict
